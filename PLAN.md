# haneng — 한/영 오타 자동 교정 유틸리티 개발 계획

## 1. 문제 정의

한국어 사용자는 한글/영문 두 가지 입력 모드를 수시로 전환하며 타이핑한다. 이때 현재 입력 모드를 착각하여:

- 영문 모드에서 한글을 입력 → `gksrmf` (의도: `한글`)
- 한글 모드에서 영문을 입력 → `ㅗ디ㅣㅐ` (의도: `hello`)

와 같은 오타가 발생하고, 사용자는 지우고 모드를 바꿔 다시 입력해야 한다. 이 앱은 **잘못된 모드로 입력된 텍스트를 감지하여 올바른 문자로 자동(또는 단축키로) 변환**해 주는 데스크톱 유틸리티다. 러시아어권의 Punto Switcher, MS 한글 IME의 "한/영 자동 전환" 기능과 동일한 문제 공간이며, 이를 **Windows / macOS / Linux 전체에서, 어떤 앱에서든** 동작하도록 만드는 것이 목표다.

## 2. 기존 솔루션과 차별점

| 솔루션 | 한계 |
|---|---|
| MS 한글 IME "한/영 자동 전환" | Windows 전용, 정확도 낮아 대부분 꺼놓음, 커스터마이징 불가 |
| Punto Switcher (Yandex) | 러시아어↔영어 전용, Windows/macOS만 |
| 웹 한영타 변환기 (예: theyt.net) | 복사-붙여넣기 필요, 실시간 아님 |
| 구름 입력기 (macOS) | IME 자체 교체가 필요, macOS 전용 |

**차별점**: 크로스플랫폼 단일 코어, IME를 교체하지 않고 기존 입력기 위에서 동작, 오픈소스로 신뢰 확보.

## 3. 요구사항

### 기능 요구사항
- **F1. 수동 변환 (MVP)**: 단축키(기본 `Ctrl/Cmd+Shift+Space`)를 누르면 마지막으로 입력한 단어(또는 선택 영역)를 한↔영 반대 모드 문자로 변환.
- **F2. 자동 감지 + 자동 변환**: 단어 경계(스페이스, 엔터, 문장부호)에서 입력된 단어를 분석해, 잘못된 모드로 판단되면 자동으로 치환. 치환 직후 `Backspace` 1회 또는 전용 단축키로 즉시 되돌리기(undo).
- **F3. 감지 민감도 설정**: 보수적(확실할 때만) ↔ 공격적(적극 변환) 슬라이더.
- **F4. 앱별 예외 설정**: 특정 앱(터미널, 게임, 원격 데스크톱, 비밀번호 관리자 등)에서 비활성화.
- **F5. 사용자 사전**: 변환하면 안 되는 단어(영문 약어 `ㅋㅋ`, `asdf` 같은 의도적 입력) 화이트리스트, 자주 틀리는 단어 학습.
- **F6. 트레이/메뉴바 UI**: 켜기/끄기 토글, 설정 창, 최근 변환 이력.
- **F7. 두벌식 우선 지원**, 이후 세벌식(390/최종) 자판 옵션.

### 비기능 요구사항
- **N1. 지연**: 키 입력 지연 체감 없음 (후킹 콜백 1ms 이내 반환, 분석은 비동기).
- **N2. 프라이버시**: 모든 처리는 로컬. 네트워크 전송 없음. 키 입력 로그를 디스크에 남기지 않음(디버그 모드 제외).
- **N3. 보안 필드 제외**: 비밀번호 입력 필드에서는 절대 동작하지 않음 (아래 8절).
- **N4. 리소스**: 상주 메모리 50MB 이하, 유휴 CPU ~0%.
- **N5. 신뢰성**: 후킹 실패/권한 회수 시 조용히 죽지 않고 사용자에게 알림.

## 4. 핵심 알고리즘 — 잘못된 모드 감지

### 4.1 문자 매핑
두벌식 자판은 QWERTY 물리 키와 1:1(쉬프트 포함 33자) 대응이다. 양방향 매핑 테이블을 만든다:
- `영→한`: `g→ㅎ, k→ㅏ, s→ㄴ, ...` 후 **자모 오토마타**로 초성-중성-종성 결합 (`gks → ㅎ+ㅏ+ㄴ → 한`). 도깨비불 현상(다음 글자 초성으로 종성이 이동)까지 표준 두벌식 오토마타 규칙 그대로 구현.
- `한→영`: 완성형 음절을 자모로 분해(유니코드 NFD + 겹자모 분해) 후 역매핑.

### 4.2 판별 스코어링
단어 `w`에 대해 두 가지 가설을 점수화한다:

1. **"현재 모드가 맞다"** 점수: `w` 자체의 언어 자연도.
2. **"모드가 틀렸다"** 점수: `w`를 반대 모드로 변환한 `w'`의 언어 자연도.

자연도 계산 (모두 로컬, 경량):
- **구조 검사** (강한 신호): 영문열이 유효한 한글 음절로 완전히 조합되는가? (`qwert`는 `ㅂㅈㄷㄱㅅ`로 음절 불가 → 한글 아님). 한글열에 `ㅗ디ㅣㅐ`처럼 조합 안 된 자모가 섞여 있는가? → 영문일 가능성 높음.
- **사전 검사**: 한국어 어휘 사전(형태소 기초형 + 조사/어미 결합 검사, 예: kiwipiepy 사전 데이터 활용) / 영어 사전 (SCOWL 등) 존재 여부.
- **문자 n-gram 빈도**: 사전에 없는 단어(신조어, 고유명사) 대비용으로 한국어 자모 bigram / 영어 letter bigram 로그확률 비교. 모델 크기 수백 KB 수준으로 충분.
- **문맥 신호**: 직전 단어들의 언어, 사용자가 방금 한/영 키를 눌렀는지 여부.

`score(w') - score(w) > threshold` 일 때만 변환. threshold가 F3의 민감도 설정과 연결된다.

### 4.3 학습 루프
사용자가 자동 변환을 undo하면 해당 단어를 로컬 예외 사전에 기록 → 같은 실수를 반복하지 않음 (F5).

## 5. 플랫폼별 통합 전략 (가장 어려운 부분)

아키텍처 원칙: **IME를 교체하지 않는다.** 기존 입력기(마이크로소프트 한글 IME, macOS 기본 한글, kime/ibus-hangul 등) 위에서 키 이벤트를 관찰하고, 필요할 때 `Backspace × n + 올바른 텍스트 주입`으로 치환한다.

| | 키 관찰 | 텍스트 치환 | 현재 IME 모드 확인 | 권한 |
|---|---|---|---|---|
| **Windows** | `SetWindowsHookEx(WH_KEYBOARD_LL)` | `SendInput` (유니코드 `KEYEVENTF_UNICODE`) | `ImmGetContext`/TSF `ITfInputProcessorProfiles` | 없음 (관리자 앱 대상 시 UIAccess 고려) |
| **macOS** | `CGEventTap` | `CGEventPost` + 유니코드 이벤트, 또는 Accessibility API로 직접 텍스트 교체 | `TISCopyCurrentKeyboardInputSource` | 손쉬운 사용(Accessibility) + 입력 모니터링 권한 |
| **Linux X11** | `XRecord` | `XTest` / uinput | IBus/Fcitx5 D-Bus API로 엔진 상태 조회 | X 세션 접근 |
| **Linux Wayland** | 전역 후킹 불가(보안 모델) → **IBus/Fcitx5 애드온**으로 구현하거나 evdev(입력 장치 직접 읽기, `input` 그룹 권한) + `uinput` 주입 | 동일 | IM 프레임워크 API | evdev 시 udev rule |

**Wayland이 최대 리스크**다. 전략: 1차로 X11 + Windows + macOS를 지원하고, Wayland는 Fcitz5/IBus 플러그인(코어 라이브러리를 FFI로 호출)으로 별도 트랙 진행. evdev 방식은 폴백으로 문서화.

치환 시 주의점:
- 조합 중인 한글(IME preedit 상태)을 건드리면 안 됨 → 단어가 커밋된 뒤(스페이스/문장부호)에만 치환.
- 치환 후 IME 모드도 함께 전환해 줌(사용자가 이어서 올바른 모드로 타이핑하도록): Windows는 `WM_IME_CONTROL`/한영 키 주입, macOS는 `TISSelectInputSource`, Linux는 IM 프레임워크 D-Bus.
- 치환 도중 사용자가 추가 입력하면 중단(race 방지: 주입 이벤트에 자체 마커 태깅).

## 6. 아키텍처와 기술 스택

```
┌─────────────────────────────────────────────┐
│  UI (트레이 아이콘, 설정 창)  — Tauri 2      │
├─────────────────────────────────────────────┤
│  haneng-core (Rust 라이브러리)               │
│   · 자모 오토마타 / 양방향 변환              │
│   · 판별 엔진 (사전 + n-gram 스코어링)       │
│   · 단어 버퍼 / undo 스택 / 사용자 사전      │
├──────────────┬──────────────┬───────────────┤
│ platform-win │ platform-mac │ platform-linux│
│ (hook+inject)│ (hook+inject)│ (X11 / IM 플러그인)│
└──────────────┴──────────────┴───────────────┘
```

- **언어: Rust** — 단일 코어를 세 플랫폼에서 재사용, 저지연·저메모리, `windows-rs`/`core-foundation`/`x11rb` 등 바인딩 성숙, IBus/Fcitx5 플러그인용 C FFI 노출 용이.
- **UI: Tauri 2** — 트레이 + 설정 창만 필요하므로 가볍고 Rust와 자연 통합. (Electron은 상주 앱으로는 과체중)
- **코어는 UI 없이도 동작** (headless 데몬) — UI는 IPC(로컬 소켓)로 데몬 제어.
- 변환 로직은 순수 함수로 분리 → 플랫폼 없이 단위 테스트 가능.

## 7. 개발 로드맵

### Phase 0 — 코어 엔진 (2~3주)
- 두벌식 양방향 변환 (자모 오토마타 포함) + 철저한 단위 테스트.
- 판별 스코어링 v1 (구조 검사 + 사전) 및 정확도 측정 하네스: 한국어/영어 말뭉치에서 오타 시뮬레이션 → precision/recall 리포트. **목표: 자동 변환 오발동(false positive) < 0.5%.**

### Phase 1 — MVP: 수동 변환 (3~4주)
- Windows + macOS에서 전역 단축키로 "마지막 단어 변환" 동작.
- 트레이 UI(켜기/끄기), 보안 필드 제외, 설치 없이 실행 가능한 빌드.
- **이 시점에 베타 배포** — 수동 변환만으로도 즉시 유용하고, 판별 엔진 없이 동작하므로 위험이 낮다.

### Phase 2 — 자동 감지 (4~6주)
- 단어 경계 자동 분석 + 자동 치환 + 원터치 undo.
- n-gram 모델, 민감도 설정, 사용자 예외 사전(undo 학습).
- IME 모드 자동 전환 동기화.

### Phase 3 — Linux + 완성도 (4~6주)
- Linux X11 지원, Fcitx5/IBus 플러그인(Wayland) 착수.
- 앱별 예외 설정, 변환 이력, 자동 업데이트.
- 세벌식 자판 옵션.

### Phase 4 — 배포/운영
- 코드 서명: Windows(EV 인증서 권장 — SmartScreen), macOS(Developer ID + notarization — 후킹 앱은 필수), Linux(AppImage/deb/rpm + AUR/Flatpak은 권한 모델 검토).
- 홈페이지, 오픈소스 공개(GitHub), 이슈 트래킹.

## 8. 보안·프라이버시 (신뢰가 곧 제품)

키보드 후킹 앱은 구조적으로 키로거와 같은 API를 쓰므로 신뢰 확보가 필수:

1. **비밀번호 필드 차단**: macOS는 `IsSecureEventInputEnabled()` 감지 시 완전 중단, Windows는 포커스 컨트롤의 `ES_PASSWORD` 스타일/UIA `IsPassword` 확인, 브라우저는 UIA/AX API로 input type 확인 가능한 범위에서 차단 + 도메인 무관 보수적 기본값.
2. **버퍼 최소화**: 마지막 1~2 단어만 메모리에 유지, 단어 경계에서 즉시 폐기. 디스크 기록 금지.
3. **네트워크 코드 자체를 미포함** (업데이트 확인 제외, 옵트인).
4. **오픈소스 + 재현 가능한 빌드**로 검증 가능성 제공.
5. 백신 오탐(휴리스틱상 키로거로 분류될 수 있음) 대비: 코드 서명, 주요 백신 벤더 화이트리스트 등록 절차 문서화.

## 9. 테스트 전략

- **코어**: 변환 왕복(round-trip) property 테스트, 도깨비불 등 오토마타 엣지 케이스, 판별 정확도 회귀 테스트(말뭉치 기반, CI에서 precision/recall 게이트).
- **통합**: OS별 VM에서 주요 앱(브라우저, 오피스, 터미널, 메신저, IDE) 대상 자동화 스모크 테스트 — 입력 주입 후 화면 텍스트 검증.
- **호환성 매트릭스**: Windows 10/11, macOS 13+, Ubuntu LTS(X11/Wayland) × 주요 IME(MS 한글, macOS 기본, ibus-hangul, fcitx5-hangul, kime).
- **성능**: 키 이벤트 콜백 지연 측정(p99 < 1ms), 장시간 상주 메모리 누수 테스트.

## 10. 주요 리스크와 대응

| 리스크 | 영향 | 대응 |
|---|---|---|
| 자동 변환 오발동 → 사용자가 기능을 꺼버림 | 치명적 | 보수적 기본 threshold, 원터치 undo, undo 학습, MVP는 수동 변환부터 |
| Wayland 전역 후킹 불가 | Linux 커버리지 | IM 프레임워크 플러그인 트랙 분리, evdev 폴백 |
| IME preedit 상태와의 충돌 (조합 중 치환) | 텍스트 깨짐 | 커밋된 단어만 치환, 주입 이벤트 마커로 race 차단 |
| macOS 권한 UX (Accessibility + 입력 모니터링 2중 허가) | 온보딩 이탈 | 권한 안내 마법사, 권한 상태 감지 및 재안내 |
| 백신/스토어의 키로거 오탐 | 배포 | 코드 서명, 오픈소스, 벤더 화이트리스트 신청 |
| 원격 데스크톱/게임 등 특수 환경 오동작 | 신뢰 | 기본 비활성 앱 목록 내장 + 앱별 설정(F4) |

## 11. 성공 지표

- 자동 변환 정확도: false positive < 0.5%, 실제 오타에 대한 recall > 90%
- 변환 후 undo 비율 < 3%
- 30일 잔존(기능 켠 상태 유지) > 60%
